# Contributing

Thank you for considering a contribution to N-Queens Visualizer.

## Development workflow

1. Fork the repository and create a branch from `main`.
2. Create and activate a Python virtual environment.
3. Install dependencies with `pip install -r requirements.txt`.
4. Make a focused change and test it locally.
5. Open a pull request with a clear description of the change.

## Pull requests

- Keep each pull request focused on one improvement.
- Explain the motivation and the approach.
- Include screenshots for visible interface changes.
- Verify that the home page and solver routes load successfully.
- Update documentation when behavior or setup changes.

## Reporting issues

When reporting a bug, include the board size, browser, steps to reproduce,
expected behavior, and actual behavior. Screenshots or console errors are useful
when available.
